对象工具类

  • MoreObjects.firstNonNull：首个参数若为null则使用第二个参数返回
  • MoreObjects.toStringHelper：用于生产toString()方法
  • Objects.hashCode：用于构建hashCode()方法
  • ComparisonChain：用于生产CompareTo()方法

函数式编程

  • Function：转换数据
  • Predicate：条件筛选

  • Supplier：提供源数据

  • Functions

    • toStringFunction：转换为toString输出
    • forMap：根据map数据决定转换结果
    • forPredicate：将Predicate转换为Function，输出为boolean
    • compose：两个Function级联成一个Function
    • constant：固定输出数据
    • forSupplier：将Supplier转换为Function
  • Predicates
    • not/and/or：关系计算
    • isNull/null：对象[不]为空的判断
    • isNull/notNull/equalTo/instanceOf/assignableFrom：对象计算
    • in：对象是否在容器中
    • compose：先转换数据，再条件筛选
    • containsPattern/contains：根据正则表达式，判断字符串是否匹配
  • Suppliers
    • compose：先获取数据，再转换数据
    • memoize：带有cache功能的Supplier，可用于延迟加载
    • memoizeWithExpiration：带有可过期cache功能的Supplier
    • ofInstance：提供固定的数据
    • synchronizedSupplier：带有同步功能的Supplier

容器

  • Collections2：通用容器工具类

    • filter：根据某个筛选条件，将容器中的数据进行过滤
    • transform：将一个容器数据转换成另一个容器数据

  • FluentIterable：多功能迭代器

    • cycle：生成环形迭代器

  • Iterables：迭代器工具类

    • removeIf：按条件删除数据
    • elementsEqual：比较两个迭代器对应的数据
    • unmodifiableIterator：封装一个不可执行修改操作的迭代器
    • size/contains/toString/toArray/addAll：基础操作
    • frequency：用于统计某个数据在容器中的出现次数
    • retainAll：将迭代器的数据与目标容器数据做合并操作
    • concat：多个迭代器连成一个迭代器
    • partition：将一个容器按照某个长度进行分割成若干个子容器
    • find/tryFind/indexOf：搜索数据
    • get/getFirst/getLast：得到数据
    • transform：转换数据

  • Lists

    • new* List：新建各种类型的list
    • asList：将数组转为list
    • reverse：数据反转
    • transform：转换数据
    • partition：将一个list按照某个长度进行分割成若干个子list

  • Sets

    • new* Set：新建各种类型的set
    • union：并集
    • intersection：交集
    • difference：减法
    • symmetricDifference：两个集合去除公共部分，即 = union - intersection

  • Maps

    • new* Map：新建各种类型的map
    • difference：对两个map进行比对
    • asMap/toMap/fromProperties：将其他容器转换为map
    • transformValues/transformEntries：转换数据
    • filterKeys/filterValues/filterEntries：数据过滤

  • 新容器

    • Multimap：允许键重复的map

      • Multimaps
      • 各种实现
        • ArrayListMultimap
        • LinkedListMultimap
        • HashMultimap：注意，不支持多次插入相同的键值对
        • TreeMultimap

    • Immutable* ：不可修改的容器

    • BiMap：双向map，可随时key/value互相转换
      • forcePut：强行插入数据，先检查是否存在相同value的数据，若存在则先清除
      • inverse：map反转
      • 各种实现：如HashBiMap
    • Table：支持二维Key的Map，可理解成数据表结构，模型为<K1,K2,V>
      + 各种实现：HashBasedTable、ArrayTable
    • Range：返回一个范围，主要用于范围比较
    • 常用工具类
    • Ordering：用于对容器数据排序，或获取最大、最小数据，检查排序等

并发

  • Monitor：用于条件等待，ReentrantLock与条件判断的合体
  • MoreExecutors：扩展后的线程池工具方法
  • ListeningExecutorService：是ExecutorService的扩展，可创建Callable的线程，并且等待线程结束后的结果
  • Futures
    • allCallback：添加future回调处理句柄FutureCallback
    • withFallback：使用FutureFallback封装一个ListenableFuture
    • transform：使用AsyncFunction对future进行转换
  • AsyncFunction：对future结果进行转换
  • FutureFallback：用于静默future异常
  • RateLimiter：流控的实现，使用信号量进行封装
    • acquire/tryAcquire：请求资源
    • setRate：设置流控值

Cache

  • CacheBuilderSpec：根据字符串（指定Cache配置）创建Cache，一般可用于配置
  • CacheBuilder：用于创建Cache
    • from：根据CacheBuilderSpec或者对应字符串创建CacheBuilder
    • expireAfterWrite/expireAfterAccess/refreshAfterWrite：用于刷新数据
    • build：创建cache
    • concurrencyLevel：指定并发级别，一般与对该map进行并发写的线程数对应
    • softValues：软引用，内存不足时可清理这部分数据
    • weakKeys/weakValues：弱引用，GC扫到则清理这部分数据
    • maximumSize：指定cache数据最大个数
    • maximumWeight：指定cache数据最大权重
    • removalListener：移除数据的监听器，用于监视数据移除的操作
    • ticker：指定纳秒时钟，默认使用System.nanoTime
    • recordStats：统计数据
  • MapMaker：用于创建ConcurrentMap
    • ConcurrencyLevel：指定并发级别，一般与对该map进行并发写的线程数对应
    • softValues：软引用，内存不足时可清理这部分数据
    • weakKeys/weakValues：弱引用，GC扫到则清理这部分数据
  • Cache
    • stats：获取统计信息
    • 几种实现：
      • LoadingCache：封装了数据加载，只需初始化时注册数据加载逻辑
      • CacheLoader：用于注册数据加载逻辑
  • CacheStats：用于统计数据

EventBus/AsyncEventBus：事件总线

  • EventBus：大总管

    • 使用方法

    1. 定义Subscriber类
      要求：
      1）有一个构造函数，参数只有一个，为EventBus，用于注册当前类，或用于后面的解注册
      2）定义事件受理方法：使用Subscribe进行注解，模型为：

    2. 定义Publisher类
      要求：
      1）有一个构造函数，参数只有一个，为EventBus，保存该句柄，用于后面的post事件

  • DeadEvent：当某个事件被发布后，没有订阅者的话EventBus会发送该事件

IO

  • Closer：用于管理多路流关闭
  • Closeables：即时关闭流

Files：文件工具类

  • newReader/asByteSource/asCharSource/newWriter/asByteSink/asCharSink：创建文件输入、输出流
  • toByteArray：将文件内容保存至byte[]
  • toString：将文件内容保存至string
  • write/append：将byte[]保存至文件
  • readLines：将文件内容按行读取
  • readFirstLine：读取文件首行
  • readBytes：将文件内容按byte[]读取
  • map：将文件内容导入bytebuffer
  • copy：将文件内容拷贝至流或者文件中
  • createTempDir：创建临时目录，目录名使用时间戳+序号生成
  • touch：创建空文件
  • createParentDirs：递归创建父目录
  • move：移动文件
  • hash：计算文件hash值
    • Hashing：返回各种HashFunction
  • getFileExtension：获取文件扩展名
  • getNameWithoutExtension：获取不带扩展名的文件名
  • simplifyPath：对文件路径归一化处理
  • isDirectory/isFile：返回是否为目录、内容的数据筛选类

Resources

  • asByteSource/asCharSource/toByteArray/toString/readLines/copy：根据URL获取资源数据
  • getResource：获取URL对象

byte

  • BaseEncoding：用于二进制编解码

net

  • HttpHeaders：http消息头通用字段名
  • MediaType：http消息头中通用的媒体类型
  • InetAddress：IP工具类
  • UrlEscapers：URL解码工具类

基本数据类型工具类

  • 位于 com.google.common.primitives

通用

  • Optional：用于包装可能出现null值的数据，对于数据传递可直接明确出来
    • absent：返回一个空对象
    • of/fromNullable：用于包装数据，前者会校验是否为空，后者当数据为空时会使用一个空对象代替
    • isPresent：判断是否存在数据
    • get：获取被包装的对象
    • or：获取被包装的对象，若对象不存在则使用入参生成对象
    • orNull：获取被包装的对象，若不存在则返回null
  • Throwables：异常工具类
    • propagate*：用于抛出需要的异常，一般用于异常封装或转换
    • getRootCause：获取异常的根因
    • getCausalChain：获取异常递归链
    • getStackTraceAsString：获取异常栈打印信息
  • Charsets：常用字符集对象列表
  • Joiner：字符串拼接
  • Splitter：字符串分割
  • Strings：字符串工具类
  • CharMatcher：字符匹配
  • Preconditons：前置条件验证
  • XmlEscapers：XML编码工具类
